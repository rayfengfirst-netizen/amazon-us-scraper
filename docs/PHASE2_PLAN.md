# 第二阶段开发计划（Phase 2）

> 目标：在保持 V1 可用性的前提下，把 Shopify 发布能力从“可用”升级为“可配置、可批量、可审计、可持续优化”。

## 1. 范围定义

### 1.1 In Scope（Phase 2）

- 发布策略配置化（每店铺）
  - 价格倍率（当前硬编码 1.7）
  - vendor 固定值（当前硬编码 `EGR Performance`）
  - SKU 规则（Amazon/eBay 前缀与提取策略）
- 文案治理
  - 4 套提示词版本化管理
  - Prompt 版本与发布日志关联（可追溯）
- 发布质量控制
  - 发布前校验（标题、价格、图片、核心字段）
  - 发布后结构化结果（渠道成功率、错误分类）
- 运营效率
  - 批量发布（按筛选条件）
  - 失败重试机制

### 1.2 Out of Scope（本阶段暂不做）

- 完整 OAuth 多租户后台（面向外部商家 SaaS）
- 复杂库存/变体矩阵（多 variant 自动生成）
- 自动化广告投放与营销闭环

## 2. 当前基线（V1）

- 详情页可手动发布到 Shopify（选店铺、状态、渠道范围）
- 支持静态 token 与 client_credentials 两种认证
- OpenAI 优化为手动触发（`use_ai`）
- 字段映射为固定规则：
  - vendor 固定值
  - tags 置空
  - SKU Amazon/eBay 双规则
  - 价格=采集价*1.7
  - 图片走 URL 直传

## 3. Phase 2 需求拆解（按优先级）

## P0（必须）

1. 发布策略配置中心（店铺级）
   - 新增字段：`price_multiplier`、`vendor_override`、`sku_mode`、`tag_policy`
   - UI：店铺设置页可编辑并立即生效
2. 发布前校验
   - 缺标题/价格/图片时阻断发布并给出可读错误
3. 日志结构化
   - `ShopifyPublishLog` 增加 `used_ai`、`prompt_version`、`payload_snapshot`

## P1（高价值）

1. 批量发布页
   - 支持按状态筛选、批量选择、并发控制
2. Prompt 版本管理
   - 模板版本号 + 发布记录关联
3. 错误治理看板
   - 401/422/429 等错误按类型聚合

## P2（增强）

1. A/B 文案策略（AI vs 本地规则）
2. 价格策略扩展（阶梯定价、最小毛利保护）
3. eBay item id 识别增强（更多样本与回归集）

## 4. 技术设计要点

- 保持“预览与实发一致”：同一套 payload builder，避免双轨逻辑漂移
- 所有外部调用（Shopify/OpenAI）必须 fail-open 或可重试，不阻断主流程
- 配置变更应具备默认值，避免影响历史店铺
- 批量任务采用后台任务队列，避免请求超时

## 5. 验收标准（Definition of Done）

- 能在不改代码的前提下，为不同店铺配置不同发布策略
- 发布前校验可拦截明显坏数据，并给用户可执行提示
- 任意一次发布都能追溯：用的店铺配置、是否启用 AI、prompt 版本、核心 payload 摘要
- 批量发布支持可观测进度与失败重试

## 6. 实施顺序建议

1. 先做配置化（P0-1）
2. 再做校验与日志（P0-2/P0-3）
3. 最后做批量与看板（P1）
